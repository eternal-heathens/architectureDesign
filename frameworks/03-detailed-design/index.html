详细设计文档输出 两名资深开发 / 一名测试
最终框架 详细设计模板

把技术架构进一步落成可开发、可测试、可上线的详细设计文档。

两名资深开发分别负责数据核心设计和服务中间件设计，一名测试负责人补齐测试矩阵、压测和故障演练， 最终输出覆盖表、索引、Redis、MQ、接口、状态机和验收门禁。

角色分工

资深开发 A

负责数据模型、表结构、索引、状态机、事务、幂等和异常恢复。

资深开发 B

负责服务拆分、接口、中间件、Redis、MQ、并发、部署和可观测性。

测试负责人

负责测试策略、用例矩阵、接口校验、故障注入、压测和验收门禁。

输出流程

输入产品方案、技术架构、约束和重点问题
开发 A数据、状态机、事务和核心流程
开发 B服务、中间件、接口和并发设计
测试测试矩阵、压测、故障演练和质疑
联合修订合并开发和测试意见
定稿形成 final-detailed-design.md
