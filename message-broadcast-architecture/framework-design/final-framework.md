# 三角色技术架构输出框架

## 1. 框架定位

该框架用于把一个技术问题拆成多角色协作流程，并输出正式技术架构方案：

```text
用户问题
  -> 技术总监定义边界和评价标准
  -> 架构师设计总体方案
  -> 技术骨干补充详细设计并提出质疑
  -> 架构师修订方案
  -> 技术总监最终裁决和定稿
```

核心价值：

- 技术总监负责收敛和裁决。
- 架构师负责构建整体架构。
- 技术骨干负责落地细化和反向质疑。

## 2. 角色配置

| 角色 | 模型/工具 | 职责 |
|---|---|---|
| 技术总监 | Codex + GPT-5.5 | 接收问题、定义边界、提出质疑、最终裁决 |
| 架构师 | Codex + GPT-5.5 | 设计总体架构、模块边界、技术选型和演进路线 |
| 技术骨干 | Claude Code + CC Switch / Codex gpt-5.5 | 详细设计、工程落地分析、风险识别、质疑架构师方案 |

## 3. 工作流

```text
1. 技术总监：问题理解与任务分派
2. 架构师：总体架构初稿
3. 技术骨干：详细设计与反向质疑
4. 架构师：基于质疑修订方案
5. 技术总监：最终评审与正式方案
```

## 5. 角色失败重试策略

- 单个角色超时或调用失败时，默认简单重试 1 次。
- 重试间隔默认 3 秒。
- 重试仍失败时，保留已完成阶段产物，并在 `run.json` 记录 `attempts`、`retries`、`retryErrors`。
- 配置错误类问题不反复重试，例如 CC Switch provider 不匹配、模型凭据缺失、命令配置缺失。


## 7. 运行命令

```bash
npm run arch -- --question "你的架构问题"
```

Mock 验证：

```bash
npm run arch:mock -- --question "你的架构问题"
```

检查 CC Switch provider：

```bash
npm run arch -- --list-cc-switch-providers
```

检查技术骨干通道：

```bash
npm run arch -- --check-senior-engineer
```
