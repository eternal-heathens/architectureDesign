# 三类方案输出触发词速查

## 1. 产品方案输出

推荐触发词：

```text
产品方案输出：
```

适用场景：你有一个产品想法、业务问题、用户需求或功能设想，希望先由 CEO 分析需求，再由产品总监输出产品化文档，最后由技术总监评估可行性和成本。

推荐输入模板：

```text
产品方案输出：
【背景】...
【目标用户】...
【核心目标】...
【当前问题】...
【约束】...
【希望输出】最终产品方案
```

别名：

```text
需求方案输出：
产品需求评审：
```

## 2. 技术架构输出

推荐触发词：

```text
架构评审：
```

适用场景：你已经有明确技术问题或系统设计问题，希望跑技术总监、架构师、技术骨干三角色，并输出正式技术架构方案。

推荐输入模板：

```text
架构评审：
【背景】...
【目标】...
【约束】...
【关注点】高并发、可靠性、成本、扩展性、故障恢复
```

别名：

```text
技术架构输出：
技术方案输出：
架构方案输出：
```

## 3. 详细设计文档输出

推荐触发词：

```text
详细设计文档输出：
```

适用场景：产品方案和技术架构已经相对明确，希望进一步细化到开发、测试、联调和上线可用的设计。

推荐输入模板：

```text
详细设计文档输出：
【产品方案】...
【技术架构】...
【重点】表结构、索引、Redis、MQ、接口、状态机、并发控制、测试方案
【希望输出】最终详细设计文档
```

别名：

```text
详细设计：
研发详细设计：
```

## 4. 角色失败重试

三个多角色方案统一采用简单重试策略：单个角色超时或调用失败时默认重试 1 次，重试间隔 3 秒；重试仍失败时保留已完成阶段产物，并记录失败角色和失败原因。
