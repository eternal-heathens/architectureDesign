# CEO / 产品总监 / 技术总监产品方案输出框架

## 1. 框架定位

该框架用于把一个初始想法、业务问题或模糊需求，整理成可确认、可评审、可验收的正式产品方案文档。

当前阶段只覆盖产品方案输出闭环：

```text
用户想法
  -> CEO 对话澄清与需求分析
  -> 产品总监产品化为 PRD
  -> 技术总监核对技术可行性、成本和风险
  -> 产品总监修订 PRD
  -> CEO 最终确认需求边界
  -> 输出最终产品方案 final-product-plan.md
```

核心价值：

- CEO 负责和用户高频对话，把想法收敛成业务目标、边界和优先级。
- 产品总监负责把需求产品化，产出结构化 PRD 和验收标准。
- 技术总监负责提前识别技术可行性、实现成本、研发风险和交付约束。
- 最终输出不是聊天记录，而是一份可以单独归档、评审和验收的正式产品方案文档。

## 2. 角色配置

| 角色 | 建议模型/工具 | 职责 |
|---|---|---|
| CEO | Codex / GPT-5.5 | 和用户对话、理解业务目标、澄清场景、确认优先级、做最终需求裁决 |
| 产品总监 | Codex / GPT-5.5 | 生成产品化需求文档、用户故事、功能边界、验收标准、版本规划 |
| 技术总监 | Codex / GPT-5.5 或本地技术评审通道 | 核对技术可行性、成本、复杂度、依赖、风险和交付周期 |

## 3. 工作流

```text
1. CEO：需求访谈与业务分析
2. 产品总监：PRD 初稿
3. 技术总监：技术可行性与成本评估
4. 产品总监：根据技术反馈修订 PRD
5. CEO：最终需求确认
6. 输出：final-product-plan.md
```

## 4. 阶段输出

### 4.1 CEO 需求访谈

```markdown
## 需求背景
## 用户原始表达
## 业务目标
## 目标用户
## 核心使用场景
## 当前痛点
## 成功指标
## 约束条件
## 不做什么
## 优先级判断
## 需要继续澄清的问题
## 分派给产品总监的任务
## 分派给技术总监的关注点
```

### 4.2 产品总监 PRD 初稿

```markdown
# 产品需求文档
## 1. 背景与目标
## 2. 用户与场景
## 3. 需求范围
## 4. 核心流程
## 5. 功能清单
## 6. 页面与交互
## 7. 数据与权限
## 8. 非功能需求
## 9. 验收标准
## 10. 版本规划
## 11. 未决问题
```

### 4.3 技术总监可行性评估

```markdown
## 技术可行性结论
## 关键技术约束
## 实现复杂度评估
## 成本评估
## 研发周期预估
## 外部依赖
## 数据与安全风险
## 性能与扩展风险
## 运维与可观测性要求
## 对 PRD 的质疑
## 建议调整项
## 是否允许继续推进需求
```

### 4.4 产品总监修订

```markdown
## 采纳的技术建议
## 未采纳的技术建议及原因
## 修订后的需求范围
## 修订后的功能清单
## 修订后的验收标准
## 版本拆分
## 技术前置条件
```

### 4.5 CEO 最终确认

```markdown
# 最终产品方案
## 1. 需求结论
## 2. 背景与目标
## 3. 目标用户
## 4. 核心场景
## 5. 业务流程
## 6. 功能范围
## 7. 页面与交互要求
## 8. 数据与权限要求
## 9. 非功能需求
## 10. 验收标准
## 11. 版本规划
## 12. 技术约束与成本结论
## 13. 不做范围
## 14. 未决问题
```

## 5. 决策规则

- CEO 优先保证业务目标清晰，不让需求在模糊状态进入后续环节。
- 产品总监优先保证需求可理解、可验证、可拆分、可验收。
- 技术总监优先保证方案可实现、成本可控、风险透明。
- 技术总监可以要求产品降级、拆期、补充约束或暂停继续推进。
- CEO 拥有最终范围裁决权，但需要保留技术总监的风险结论。

## 6. 角色失败重试策略

- 单个角色超时或调用失败时，默认简单重试 1 次。
- 重试间隔默认 3 秒。
- 重试仍失败时，保留已完成阶段产物，并标记失败角色和失败原因。
- 配置错误类问题不反复重试，例如模型凭据缺失、provider 不匹配、命令不存在。

## 7. 输出目录约定

```text
outputs/<timestamp>/
  00-user-request.md
  01-ceo-requirement-analysis.md
  02-product-director-prd-draft.md
  03-technical-director-feasibility.md
  04-product-director-prd-revision.md
  05-final-product-plan.md
```

## 8. 最终产品方案质量门禁

`05-final-product-plan.md` 必须满足：

- 业务目标明确。
- 目标用户明确。
- 核心场景明确。
- 功能范围明确。
- 不做范围明确。
- 验收标准明确。
- 非功能需求明确。
- 成本与技术风险有结论。
- 未决问题被单独列出，不混在已确认需求里。

```bash

```
